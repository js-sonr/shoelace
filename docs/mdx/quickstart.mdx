## Getting Started

<Steps>
<Step>

### Choose Your Installation Method

Nebula can be loaded via CDN or installed locally through npm. The CDN approach is the easiest way to get started.

<Tabs items={['CDN (Recommended)', 'npm', 'Bundler']}>
  <Tab value="CDN (Recommended)">
    ```html
    <!-- Autoloader (Recommended) -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@sonr.io/nebula@latest/dist/themes/light.css" />
    <script type="module" src="https://cdn.jsdelivr.net/npm/@sonr.io/nebula@latest/dist/shoelace-autoloader.js"></script>
    ```
    
    The autoloader watches for unregistered Nebula elements and loads them on demand.
  </Tab>
  <Tab value="npm">
    ```bash
    npm install @sonr.io/nebula
    ```
    
    Then include the files in your HTML:
    ```html
    <link rel="stylesheet" href="/node_modules/@sonr.io/nebula/dist/themes/light.css" />
    <script type="module" src="/node_modules/@sonr.io/nebula/dist/shoelace.js"></script>
    ```
  </Tab>
  <Tab value="Bundler">
    ```javascript
    import '@sonr.io/nebula/dist/themes/light.css';
    import '@sonr.io/nebula/dist/components/button/button.js';
    import '@sonr.io/nebula/dist/components/icon/icon.js';
    import { setBasePath } from '@sonr.io/nebula/dist/utilities/base-path.js';
    
    // Set the base path for assets
    setBasePath('/node_modules/@sonr.io/nebula/dist');
    ```
  </Tab>
</Tabs>

</Step>

<Step>

<Step>

### Configure Your Theme

Nebula supports both light and dark themes. You can set the theme by adding a class to your HTML element:

```html
<!-- Light theme (default) -->
<html>

<!-- Dark theme -->
<html class="sl-theme-dark">

<!-- Auto theme based on system preference -->
<link
  rel="stylesheet"
  media="(prefers-color-scheme:light)"
  href="https://cdn.jsdelivr.net/npm/@sonr.io/nebula@latest/dist/themes/light.css"
/>
<link
  rel="stylesheet"
  media="(prefers-color-scheme:dark)"
  href="https://cdn.jsdelivr.net/npm/@sonr.io/nebula@latest/dist/themes/dark.css"
  onload="document.documentElement.classList.add('sl-theme-dark');"
/>
```

</Step>

<Step>

### Add Blockchain-Specific Icons

Nebula includes a comprehensive set of cryptocurrency icons. Enable them by registering the crypto icon library:

```javascript
import { registerIconLibrary } from '@sonr.io/nebula/dist/utilities/icon-library.js';

// Register cryptocurrency icons
registerIconLibrary('crypto', {
  resolver: name => `https://cdn.jsdelivr.net/npm/@sonr.io/nebula@latest/dist/assets/crypto-icons/${name}.svg`
});
```

Then use them in your components:

```html
<nu-icon library="crypto" name="btc"></nu-icon>
<nu-icon library="crypto" name="eth"></nu-icon>
<nu-icon library="crypto" name="usdc"></nu-icon>
```

</Step>
</Steps>

## Framework Integration

<Tabs items={['React', 'Vue', 'Angular']}>
  <Tab value="React">
    Nebula provides React wrappers for all components:
    
    ```tsx
    import { SlButton, SlIcon, SlCard } from '@sonr.io/nebula/dist/react';
    
    function WalletCard() {
      return (
        <SlCard>
          <SlIcon library="crypto" name="metamask" slot="image"></NuIcon>
          <strong>MetaMask Wallet</strong>
          <SlButton variant="primary">Connect</NuButton>
        </NuCard>
      );
    }
    ```
  </Tab>
  
  <Tab value="Vue">
    Use Nebula components directly in Vue templates:
    
    ```vue
    <template>
      <nu-card>
        <nu-icon library="crypto" name="metamask" slot="image"></nu-icon>
        <strong>MetaMask Wallet</strong>
        <nu-button variant="primary" @click="connectWallet">
          Connect
        </nu-button>
      </nu-card>
    </template>
    
    <script>
    export default {
      methods: {
        connectWallet() {
          // Connection logic
        }
      }
    }
    </script>
    ```
  </Tab>
  
  <Tab value="Angular">
    Add CUSTOM_ELEMENTS_SCHEMA to your module:
    
    ```typescript
    import { CUSTOM_ELEMENTS_SCHEMA, NgModule } from '@angular/core';
    
    @NgModule({
      schemas: [CUSTOM_ELEMENTS_SCHEMA]
    })
    export class AppModule { }
    ```
    
    Then use in templates:
    
    ```html
    <nu-card>
      <nu-icon library="crypto" name="metamask" slot="image"></nu-icon>
      <strong>MetaMask Wallet</strong>
      <nu-button variant="primary" (click)="connectWallet()">
        Connect
      </nu-button>
    </nu-card>
    ```
  </Tab>
</Tabs>

## Common Patterns

### Wallet Connection Button

```html
<nu-button variant="primary" size="medium">
  <nu-icon library="crypto" name="wallet" slot="prefix"></nu-icon>
  Connect Wallet
</nu-button>
```

### Transaction Status Alert

```html

  <nu-icon slot="icon" name="check2-circle"></nu-icon>
  <strong>Transaction Confirmed!</strong><br />

</nu-alert>
```


## Frequently Asked Questions

<Accordions type="single" collapsible>
  <Accordion title="How do I customize component styles?" id="custom-styles">
    Nebula components can be customized using CSS custom properties (design tokens) and CSS parts:
    
    ```css
    nu-button {
      --nu-input-height-medium: 48px;
      --nu-input-font-size-medium: 18px;
    }
    
    nu-button::part(base) {
      border-radius: 9999px;
    }
    ```
  </Accordion>
  
  <Accordion title="Can I use Nebula with TypeScript?" id="typescript">
    Yes! Nebula is written in TypeScript and provides full type definitions. When using npm/bundler installation, types are automatically included:
    
    ```typescript
    import type { SlButton } from '@sonr.io/nebula';
    
    const button = document.querySelector<SlButton>('nu-button');
    button?.click();
    ```
  </Accordion>
  
  <Accordion title="How do I lazy load components?" id="lazy-loading">
    When using the autoloader (CDN), components are automatically lazy loaded. For bundler setups, you can use dynamic imports:
    
    ```javascript
    // Lazy load when needed
    import('@sonr.io/nebula/dist/components/dialog/dialog.js').then(() => {
      // nu-dialog is now available
      const dialog = document.querySelector('nu-dialog');
      dialog.show();
    });
    ```
  </Accordion>
  
  <Accordion title="Are components accessible?" id="accessibility">
    Yes! All Nebula components are built with accessibility in mind:
    - Proper ARIA attributes
    - Keyboard navigation support
    - Screen reader compatibility
    - Focus management
    - High contrast mode support
  </Accordion>
  
  <Accordion title="How do I handle Web3 integration?" id="web3">
    Nebula components are designed to work seamlessly with Web3 libraries. Here's an example with ethers.js:
    
    ```javascript
    import { ethers } from 'ethers';
    
    const connectButton = document.querySelector('nu-button.connect-wallet');
    connectButton.addEventListener('click', async () => {
      connectButton.loading = true;
      
      try {
        const provider = new ethers.BrowserProvider(window.ethereum);
        await provider.send("eth_requestAccounts", []);
        connectButton.loading = false;
        connectButton.variant = 'success';
        connectButton.textContent = 'Connected';
      } catch (error) {
        connectButton.loading = false;
        // Handle error
      }
    });
    ```
  </Accordion>
</Accordions>
