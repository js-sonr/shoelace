According to [The A11Y Project](https://www.a11yproject.com/posts/2013-01-11-how-to-hide-content/), "there are real world situations where visually hiding content may be appropriate, while the content should remain available to assistive technologies, such as screen readers. For instance, hiding a search field's label as a common magnifying glass icon is used in its stead."

Since visually hidden content can receive focus when tabbing, the element will become visible when something inside receives focus. This behavior is intentional, as sighted keyboard user won't be able to determine where the focus indicator is without it.

## Examples

### Links That Open in New Windows

In this example, the link will open a new window. Screen readers will announce "opens in a new window" even though the text content isn't visible to sighted users.

<Tabs items={['Preview', 'Web Component', 'React']}>
  <Tab value="Preview">
    ```tsx
    <a href="https://example.com/" target="_blank">
      Visit External Page
      <nu-icon name="box-arrow-up-right"></nu-icon>
      <nu-visually-hidden>opens in a new window</nu-visually-hidden>
    </a>
    ```

### Content Conveyed By Context

Adding a label may seem redundant at times, but they're very helpful for unsighted users. Rather than omit them, you can provide context to unsighted users with visually hidden content that will be announced by assistive devices such as screen readers.

<Tabs items={['Preview', 'Web Component', 'React']}>
  <Tab value="Preview">
    ```tsx
    <nu-card style="width: 100%; max-width: 360px;">
      <header>
        <nu-visually-hidden>Personal Info</nu-visually-hidden>
      </header>
      <nu-input label="Name" style="margin-bottom: .5rem;"></nu-input>
      <nu-input label="Email" type="email"></nu-input>
    </nu-card>
    ```
  </Tab>
